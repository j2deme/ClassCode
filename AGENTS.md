# Agentes

- Para cada folder que represente una unidad o tema, asegurar que contiene un README.md claro y detallado.
- Este README.md debe incluir:
  - Nombre de la asignatura y unidad ("Algoritmia y Estructura de Datos - Unidad X").
  - Contenidos: Listado de archivos principales y su propósito, sin entrar en excesivo detalle.
  - Requisitos: Instrucciones claras sobre cómo ejecutar ejemplos o demos.
  - ¿Cómo usarlo?: Pasos concretos para ejecutar el código, incluyendo comandos de terminal si es necesario.
  - Notas y retos: Sugerencias para profundizar o mejorar el material, como enfocar el estudio en archivos específicos o implementar métodos adicionales.
- Cada sección debe estar claramente marcada con un encabezado (##).
- Evitar secciones innecesarias o redundantes, manteniendo el README.md conciso pero informativo.
- Asegurar que el lenguaje sea claro y accesible para estudiantes que puedan no estar familiarizados con todos los términos técnicos.
- Mantener un tono didáctico y orientado al aprendizaje.
- Revisar que no haya instrucciones contradictorias o confusas en las secciones de uso y notas.
- Verificar que los comandos de terminal sean correctos y estén actualizados para el entorno Python recomendado.
- Asegurar que las referencias a versiones de Python o librerías externas sean precisas y relevantes para el material presentado.
- Asegurar que cada que se genera un README.md, este siga la estructura y lineamientos definidos aquí.
- Actualizar el README principal del repositorio para reflejar los cambios en las unidades o temas.
